# CLAUDE.md

Project context for Claude Code. Read before making changes.

## What this is
Master's thesis: a daily event study of whether Chinese monetary policy
shocks move ASEAN-5 (Indonesia, Malaysia, Philippines, Singapore, Thailand)
exchange rates against the US dollar. Preferred baseline is a country
fixed-effects model on the 2008--2020 sample, [0,+1] event window, one-year
shock measure (shock_1y), standard errors clustered by date.

## Hard rules (do not break)
- **Never write to `data-raw/`.** It holds original source files. Read only.
- **Everything in `data-clean/` is script output.** Never hand-edit it; it is
  regenerated by the R scripts.
- **Scripts run in numbered order.** `R/00`–`R/05`, orchestrated by
  `run_all.R`. Each script begins with `source("R/00_setup.R")`.
- **Paths come from `00_setup.R`** (the `paths` list) and the sample dates
  from `SAMPLE_START` / `SAMPLE_END`. Do not hard-code paths or dates
  elsewhere; do not call `setwd()`.
- Open the project through the `.Rproj` file so the working directory is the
  project root.

## Pipeline
- `R/00_setup.R`        packages, paths, constants
- `R/01_load_policy.R`  raw shocks -> policy_shocks_main.csv, policy_shocks_broad.csv
- `R/02_clean_fx.R`     raw FX workbook -> fx_asean5_clean.csv, fx_returns_wide.csv
- `R/03_build_panel.R`  -> fx_panel.csv, reg_data0.csv, reg_data01_main.csv
- `R/04_estimate.R`     -> output/models.rds
- `R/05_tables_figures.R` -> output/tables/*, output/figures/*

## How to verify a change
Run `source("run_all.R")` top to bottom with no errors, then confirm the
numbers in `output/tables/` still match the committed draft results.

## Known things to keep an eye on
- `datetime` in the raw shock file has no seconds; parse with `ymd_hm`.
- The FX workbook has a title row above the header: read with `skip = 1`.
